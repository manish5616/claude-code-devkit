## Architecture overview (narrated from the facts)

### Purpose
**Click** is a Python library for building command-line interfaces declaratively —
you compose commands and options with decorators, and Click handles parsing, help
text, prompting, and error reporting.

### Structure
The whole library lives in one package, `src/click/`. The symbol counts make the
shape obvious: **`core.py` is the heart** (146 symbols — 25% of the entire library),
with `types.py` (parameter type conversion) the clear second pillar. Supporting
modules handle cross-cutting concerns — `_compat.py` (Python/OS portability),
`termui.py`/`_termui_impl.py` (terminal UI: prompts, progress bars),
`formatting.py` (help layout), and `testing.py` (a test harness for CLIs).

### The core model (from `core.py`)
The class method-counts reveal the central abstractions:

- **`Context` (32 methods)** — the richest class. It carries invocation state down
  the command tree (params, parent links, resource cleanup). Its size signals that
  *state management*, not parsing, is Click's hardest problem.
- **`Command` (26)** and **`Group` (19)** — a command, and a command that contains
  subcommands. This pair is the Composite pattern that gives Click its nested
  `git`-style sub-command structure.
- **`Parameter` (21)** with subclasses **`Option` (13)** and **`Argument` (9)** —
  the inheritance hierarchy for everything a command can accept.

Top-level helpers like `iter_params_for_processing` and `augment_usage_errors`
confirm that `core.py` also owns the *processing pipeline*, not just the data model.

### Where to start reading
For a newcomer, the facts suggest this order:
1. `core.py` → `Command`, `Group`, `Context` (the model + execution)
2. `core.py` → `Parameter`/`Option`/`Argument` (inputs)
3. `types.py` (how raw strings become typed values)
4. `decorators.py` (the `@click.command()` API users actually touch)

### Maintenance signal
`core.py`'s blame story shows active, recent maintenance (latest change 2026-05-19)
concentrated among a few core maintainers — a healthy sign for a foundational file.
